LOCAL CI-CD PLAN — GITHUB ACTIONS + COLIMA
==========================================
Date: 2026-03-26 (updated: pod-based deployment)

HOW IT ALL FITS TOGETHER
-------------------------
Your GitHub Actions pipelines:
  - Build each app
  - Run tests
  - Push Docker images to GitHub Container Registry (ghcr.io)

Two deployment options are available:
  Option A — docker-compose  (simple, single container per app)
  Option B — Kubernetes pods (3 replicas per app, recommended)

Image locations:
  Backend          → ghcr.io/rishabhgupta9695/shopizer:latest
  Frontend (combined) → ghcr.io/rishabhgupta9695/shopizer-frontend:latest
    (single nginx serving both Angular admin at /admin and React shop at /shop)

Note: Admin and Shop are combined into one nginx image to reduce pod count.
      The combined image is built from nginx/Dockerfile in shopizer-infra repo.


PART 1 — ONE-TIME SETUP (both options)
========================================

Step 1: Install tools
----------------------
  brew install colima docker docker-compose kubectl

Step 2: Authenticate with GitHub Container Registry
-----------------------------------------------------
  2a. Go to: https://github.com/settings/tokens
  2b. Generate new token (classic) with read:packages scope
  2c. Run:
      echo "YOUR_TOKEN" | docker login ghcr.io -u rishabhgupta9695 --password-stdin


OPTION A — DOCKER-COMPOSE (simple)
=====================================

Start Colima:
  colima start --cpu 4 --memory 6 --disk 60

Deploy:
  cd /Users/rishabhgupta/Documents/shopizer-infra
  docker-compose pull && docker-compose up -d

Access:
  Backend   →  http://localhost:8080/swagger-ui.html
  Admin     →  http://localhost:4200
  Shop      →  http://localhost:3000

Update after new CI build:
  docker-compose pull && docker-compose up -d

Stop:
  docker-compose down
  docker-compose down -v   ← also wipes MySQL data


OPTION B — KUBERNETES PODS (recommended, 3 replicas each)
===========================================================

Start Colima with Kubernetes:
  colima start --cpu 4 --memory 6 --disk 60 --kubernetes

Create pull secret for ghcr.io:
  kubectl create secret docker-registry ghcr-secret \
    --docker-server=ghcr.io \
    --docker-username=rishabhgupta9695 \
    --docker-password=YOUR_GITHUB_TOKEN \
    --namespace=shopizer

Deploy all:
  cd /Users/rishabhgupta/Documents/shopizer-infra
  kubectl apply -f k8s/namespace.yaml
  kubectl apply -f k8s/mysql.yaml
  kubectl apply -f k8s/shopizer.yaml
  kubectl apply -f k8s/shopizer-frontend.yaml

Verify pods (expect 7 total):
  kubectl get pods -n shopizer

  mysql              × 1
  shopizer           × 3
  shopizer-frontend  × 3  (nginx serving admin + shop)

Access:
  Backend   →  http://localhost:8080/swagger-ui.html
  Admin     →  http://localhost:30200/admin
  Shop      →  http://localhost:30200/shop

Update after new CI build:
  kubectl rollout restart deployment/shopizer -n shopizer
  kubectl rollout restart deployment/shopizer-frontend -n shopizer

Tear down:
  kubectl delete namespace shopizer


PART 2 — BUILDING THE COMBINED FRONTEND IMAGE
===============================================
Admin and Shop are served by a single nginx image.
It must be rebuilt whenever either frontend app changes.

Trigger the build manually:
  GitHub → shopizer-infra repo → Actions → "Build Frontend Image" → Run workflow

Or it triggers automatically on push to main when nginx/ files change.

The Dockerfile (nginx/Dockerfile) pulls both CI images and combines them:
  /usr/share/nginx/admin  ← Angular admin
  /usr/share/nginx/shop   ← React shop


PART 3 — COMPLETE CI-CD FLOW
==============================

  1. Push code to shopizer / shopizer-admin / shopizer-shop repos
         ↓
  2. GitHub Actions builds, tests, pushes image to ghcr.io
         ↓
  3. For frontend changes: trigger "Build Frontend Image" in shopizer-infra
         ↓
  4. Deploy update:
       docker-compose:  docker-compose pull && docker-compose up -d
       kubernetes:      kubectl rollout restart deployment/... -n shopizer
         ↓
  5. New version live at localhost


PART 4 — VIEW LOGS
===================
docker-compose:
  docker-compose logs -f
  docker-compose logs -f shopizer

kubernetes:
  kubectl logs -f deployment/shopizer -n shopizer
  kubectl logs -f deployment/shopizer-frontend -n shopizer


PART 5 — TROUBLESHOOTING
==========================

Problem: "unauthorized" when pulling images
Fix:     Re-run docker login from Part 1 Step 2

Problem: "Cannot connect to Docker daemon"
Fix:     colima start

Problem: Port 3306 already in use
Fix:     brew services stop mysql

Problem: Not enough memory
Fix:     colima stop && colima start --cpu 4 --memory 8 --disk 60

Problem: Pods stuck in ImagePullBackOff
Fix:     kubectl describe pod <pod-name> -n shopizer
         Usually means ghcr-secret is missing or token expired
         Re-create the secret from Part 1
